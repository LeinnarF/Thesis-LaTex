\section{Results and Discussion}

This section presents the results of the study, focusing on the development of a mobile itinerary generator and optimizer for tourist destinations in the province of Bulacan. The results follow a comparative analysis between selected algorithms to solve the Traveling Salesman Problem (TSP). It includes the discussion about the developed system alongside the features available with the application. The reliability and quality of the application is also measured under user acceptability, and software quality assessments. The results of the study provides insights into how modern planning methods affect the tourism sector. It enables policy makers to create more suitable methods of promotion and explore which aspect of the sector can be changed to improve the quality of tourism and provide a better experience for the tourists.

\subsection{Route Optimization Algorithm Selection}
This section shows how the selected metaheuristic algorithms perform when solving TSP. The burma14, bays29, eil51, and berlin52 instances from TSPLIB library were used to test the algorithms. The study aimed to determine which route optimization algorithm would best serve Lakad by evaluating its ability to minimize travel distance and runtime efficiency. The researchers executed each algorithm 10 times to observe its stochastic nature and record the average results for analysis.

Each algorithm has their specific parameters to ensure fair comparison. The researchers executed the algorithms 30 times for each TSP instance to establish a statistical validity, which resulted in average performance metrics that they used for their analysis. The parameter settings were selected based on preliminary experiments and studied literature recommendations.

The solution quality of each algorithm is measured by its average relative error (\%) when compared to the optimal solution for each TSP instance. While the runtime efficiency is computed focusing on mobile environment which requires quick response as users need it instantly when optimizing their itinerary.

Table \ref{tab:rank} displays the scores and rankings for each individual instance. Both criteria received equal weight \( w_{i} \) which were both equal to 0.5. The SAW scores showed that Simulated Annealing (SA) prevail as the best algorithm since it achieved optimal solution with efficient runtime performance. GA achieved the highest accuracy but its increased runtime with larger problem sizes resulted in a consistently undesirable SAW score which positioned it in 6th place. EHO performed effectively on small instances but its accuracy declined when handling larger cases which made it unsuitable for complex itineraries.

Given the results, the Simulated Annealing (SA) was the selected route optimization for Lakad. It provides a good trade-off between solution quality and runtime efficiency, ensuring that users receive optimized itineraries in a reasonable time frame, enhancing the overall user experience of the Lakad application. The result follow the argument of \textcite{Wadi2025} that swarm-based or heuristic algorithms with fast runtimes are better suited for resource-constrained mobile environments compared to the ones with better quality but slow execution, which also risks critical overuse of resources in mobile phones.

\subsection{The Developed System}
The developed system, Lakad, was developed as both a place explorer and itinerary planner with route optimization capabilities. The features integrated in the developed system aimed to fill the gaps where previous systems struggled to fill. The developed system also proved capable of contributing to the ongoing efforts of \parencite{DOT2023} to promote a distinct personal experience for tourists.
With this in mind, Lakad has five key features namely; Tourist Spot Exploration which includes Tourist Spot Information, and Reviews and Ratings; the Personalized Itinerary Generation feature which contains the Generated Itinerary; the Itinerary Optimization which utilizes the selected algorithm for optimization; the Itinerary Management feature that lets users manage itineraries, including an option to create more manually or automatically; and the Itinerary Navigation feature which serve as the guide for traversing given itinerary. Alongside the core features, the application also has an admin mode for a more efficient management of the application's content by the administrators.

\subsection{System Evaluation}
This section presents evaluation results of Lakad in terms of user acceptability and software quality of the system. The acceptability assessment uses end-user responses for constructs found in the Technology Acceptance Model (TAM) while the software quality is evaluated using responses of experts on a survey that is based on the standards set by ISO/IEC 25010:2023. The collected data was processed and analyzed in a Jupyter Notebook with the use of Python Libraries such as Pandas, Matplotlib, and NumPy. The results are shown through mean scores and standard deviations.
